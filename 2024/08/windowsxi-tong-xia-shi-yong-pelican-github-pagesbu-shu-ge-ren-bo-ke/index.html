Windows系统下使用Pelican + GitHub Pages部署个人博客

Python 2024-08-15

安装Pelican并尝试使用

首先创建一个新的Python项目，并在该路径下打开终端。

执行以下命令安装Pelican，并生成一个简单的网站架构：

.venv\Scripts\pip install pelican[markdown]
.venv\Scripts\python -m pip install --upgrade pip
.venv\Scripts\pelican-quickstart

此时需要回答一系列问题帮助创建文件，其中默认语言若选择简体中文，在之后的过程中会发生错误，其余问题可按个人需求回答：

> Where do you want to create your new web site? [.]
> What will be the title of this web site? 梅岭随记
> Who will be the author of this web site? 挪希
> What will be the default language of this web site? [Chinese (Simplified)] en
> Do you want to specify a URL prefix? e.g., https://example.com   (Y/n) n
> Do you want to enable article pagination? (Y/n) n
> What is your time zone? [Europe/Rome]
> Do you want to generate a tasks.py/Makefile to automate generation and publishing? (Y/n) n
Done. Your new project is available at C:\Users\Vergica\PycharmProjects\Blog

新建一个简单的博客用以测试。注意须指定文件编码为utf-8，但代码中不应出现连字号：

echo "Title: My First Review" "Date: 2010-12-03 10:20" "Category: Review" "Following is a review of my favorite mechanical keyboard." | out-file content\review.md -encoding utf8

生成静态文件并监听本地8000号端口：

.venv\Scripts\pelican content
.venv\Scripts\pelican --listen

若操作得当，此时http://127.0.0.1:8000可正常访问，显示为初始博客界面。

推送至GtiHub

首先，应确保电脑上安装了Git，接下来的操作会在Git Bash中进行。

在博客目录下的output文件夹打开Git Bash。初始化Git仓库，并配置用户信息：

git init
git config user.name "vergica" --global
git config user.email "1234567890@qq.com" --global

在GitHub创建一个新的公共仓库，复制SSH链接，添加仓库：

git remote add origin git@github.com:vergica/vergica.github.io.git

现在，本地仓库就和远程仓库建立好了对应的联系。

然后我们需要生成一对密钥，可选的选项大多选择默认即可：

ssh-keygen

输出的末尾为randomart image，表示密钥生成成功。打开密钥所在目录，复制公钥内容。

浏览器访问SSH and GPG keys，选择新建SSH key，填写公钥并确认。

最后重新回到本地，完成推送：

git add .
git commit -m "init"
git push -u origin master

前往打开仓库的GitHub Pages，如https://github.com/vergica/vergica.github.io/settings/pages ，即完成了初步部署。
